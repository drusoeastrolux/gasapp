### Database Tables
Created two tables in Supabase:

**`gas_stations`**
- Stores station info fetched from Google Places API
- Columns: `station_id`, `station_name`, `station_address`, `station_lat`, `station_lng`, `station_created_at`
- RLS disabled (public data, no need to restrict)

**`favorites`**
- Stores user-saved stations
- Columns: `favorite_id`, `user_id`, `station_id`, `favorite_created_at`
- Foreign keys to `auth.users` and `gas_stations` with cascade delete
- RLS enabled — users can only access their own favorites

### Backend (server/index.js)
- Installed `@supabase/supabase-js` on the server
- After a fresh Google Places API fetch (cache miss), stations are upserted into `gas_stations`
- On Redis cache hit, no Supabase write — data already exists, not worth the extra call
- Decision: upsert only on fresh Google fetch. Cache miss auto-heals Supabase within 20 min if ever wiped

## Problems Faced

**RLS blocking upserts**
The anon key couldn't insert into `gas_stations` because Supabase enables RLS by default. Fixed by disabling RLS on that table since station data is public.

**Email confirmation redirect failing locally**
Supabase's confirmation email links redirect to the production URL, which doesn't exist locally. Fixed by disabling email confirmation in the Supabase dashboard (re-enable before going live).

**SIGN IN button overlapping LIST/MAP toggle**
The LIST/MAP toggle was `fixed` positioned in the top-right, sitting on top of the SIGN IN button. Fixed by moving the toggle into the header row alongside the auth button.

**Favorites only showing current city's stations**
The favorites toggle was filtering the current station list, so stations from other cities weren't visible. Fixed by fetching all favorited stations directly from Supabase (`favorites` joined with `gas_stations`) instead of filtering the local list.

**Favorites showing no prices**
Prices aren't stored in the database — they're generated by `seedPrice()` on the server. The favorites view had no price to pull. Fixed by replicating the same deterministic `seedPrice()` function on the frontend.
